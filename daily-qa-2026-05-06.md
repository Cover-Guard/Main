## Coverage gaps closed (5 new probes + 1 redundancy guard carried)

`Main/scripts/qa/smoke-test.ts` (730 lines):

1. **`OPTIONS /api/properties/search` from disallowed origin → 204, no
   ACAO echo** — pins the CORS deny path. The 2026-05-05 run mentioned
   this as a tomorrow-target. A regression that loosens
   `isOriginAllowed` (e.g. catch-all `return true`, accidental `*`
   fallback) would silently expose the API to CSRF from any origin —
   **without changing the smoke-test PASS count**, since no existing
   probe tests origins.
2. **`OPTIONS /api/properties/search` from allowed origin → 204, ACAO
   echoed (and not `*`)** — pins the CORS happy path. A regression that
   breaks the `CORS_ALLOWED_ORIGINS` parser (e.g. drops the
   `.map(trim)`, accidentally swaps to `*` for `credentials:true`) would
   silently deny legitimate browsers in production. Also pins the
   `credentials:true` invariant: `*` is invalid alongside credentials,
   so the smoke probe asserts ACAO !== `*`.
3. **`POST /api/properties/geocode` with `placeId=""` → 400** — pins
   zod's `.min(1)` lower bound. Different code path from the empty-body
   case (the empty-body probe hits zod's "field absent" branch; this
   one hits `.min(1)` validation). Catches a regression that drops
   `.min(1)` from the schema.
4. **`POST /api/properties/geocode` with `placeId>300 chars` → 400** —
   pins zod's `.max(300)` upper bound. The 2026-05-01 morning run added
   the empty-body 400 probe but a length-cap regression wouldn't have
   been caught.
5. **`HEAD /api/properties/<bogus>/report.pdf` → 404 (NOT 401)** —
   completes the GET/HEAD × valid/bogus matrix on `/report.pdf`. The
   2026-05-04 run pinned `GET + bogus → 404`, the 2026-05-05 run
   pinned `HEAD + valid → 401`. Today's probe pins `HEAD + bogus → 404`,
   closing the matrix. Catches a regression where the param middleware
   special-cases GET (e.g. checks `req.method` before running
   `ensurePropertyId`) and lets HEAD bypass the resolution check —
   either leaking existence info (200 with no body) or shape info (401
   confirming the id is valid-shaped).
