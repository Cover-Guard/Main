## Coverage gaps closed (5 new probes + extended file-integrity guard)

`Main/scripts/qa/smoke-test.ts` (897 lines):

1. **`GET /api/properties/suggest` Cache-Control includes `s-maxage=300`
   and `public`** — pins the CDN cache directive on the suggest
   endpoint. The handler calls `setCacheHeaders(res, 300, 60)` which
   emits `public, s-maxage=300, stale-while-revalidate=60`. A regression
   that drops the call would silently triple upstream load on Vercel
   Edge by turning off CDN caching for typeahead — the highest-volume
   endpoint after `/health`. **Without this pin, no smoke probe checks
   any Cache-Control header.**
2. **`GET /api/properties/suggest` exposes standard `RateLimit-Limit` +
   `RateLimit-Remaining`** — pins the `standardHeaders:true` /
   `legacyHeaders:false` config in `makeLimiter()`. A regression that
   flips standardHeaders to false (or accidentally re-enables
   legacyHeaders) would silently break clients that negotiate retry
   timing from these headers — particularly mobile + frontend retry
   helpers that key off the RFC draft headers.
3. **`OPTIONS /api/auth/me` from allowed origin → 204, ACAO echoed
   (and not `*`)** — second mounted surface beyond
   `/api/properties/search`. The cors() middleware is mounted globally
   today, so behavior is identical, but a future per-router cors
   override (e.g. `app.use('/api/auth', cors({origin:'*'}), authRouter)`)
   could silently change contract on one surface only. As more
   sensitive routers are mounted (e.g. `/api/admin`, `/api/billing`),
   each should get its own preflight pin.
4. **`GET /api/properties/:id` Cache-Control includes `s-maxage=1800`
   and `public`** — pins the 30-min CDN cache on property detail. The
   property-detail endpoint fans out to ATTOM (commercial API with rate
   limits); the CDN cache is the primary defense against rate-limit
   exhaustion under load.
5. **`GET /api/properties/:id/risk` Cache-Control includes
   `s-maxage=7200` and `public`** — pins the 2-hour CDN cache on the
   risk profile (default branch; `?refresh=true` would emit no-cache).
   Risk computation fans out to FEMA NFHL, Cal Fire FHSZ, USGS NSHM,
   and FBI UCR — four separate upstream APIs. Without the CDN cache,
   each /:id/risk request hits all four. The 2-hour cache is sized
   against the rate-limit floor of the slowest upstream (USGS).
