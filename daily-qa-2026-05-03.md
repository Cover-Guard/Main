## Gaps found and closed

Nine probes were missing from `scripts/qa/smoke-test.ts`. Every one of them
covers a verb that already exists in production code today; if any of them
silently lost `requireAuth` (or, for the webhook, signature checking), the
regression would ship without a test failing. All nine are now in the
smoke script.

| # | Probe | Why it matters |
|---|---|---|
| 1 | `PATCH /api/clients/:id` → 401 | `clientsRouter.use(requireAuth)` gates all four verbs; only GET + POST were probed. |
| 2 | `DELETE /api/clients/:id` → 401 | same surface, same risk. |
| 3 | `PATCH /api/deals/:id` → 401 | `dealsRouter.use(requireAuth)`; only GET + POST were probed. |
| 4 | `DELETE /api/deals/:id` → 401 | same. |
| 5 | `DELETE /api/properties/:id/save` → 401 | POST was probed on 2026-05-02; DELETE wasn't. |
| 6 | `PATCH /api/properties/:id/checklists/:checklistId` → 401 | per-row mutations on a brand-new surface; never probed. |
| 7 | `DELETE /api/properties/:id/checklists/:checklistId` → 401 | same. |
| 8 | `GET /api/properties/<bogus>` → 404 | only `/walkscore` on a bogus id was probed; the bare detail handler hits a different `:id` middleware branch. |
| 9 | `POST /api/stripe/webhook` (no `stripe-signature`) → 400 | Stripe webhook is unauth by design; signature is the only protection. Was never probed. A regression that accepts unsigned payloads lets anyone forge subscription events. |

## Suggested PR body

```markdown
## What

Daily smoke + QA pass for 2026-05-03. Adds 9 missing smoke probes and a
unit test pinning the new contracts.

## Why

Walking the route surface mounted in `apps/api/src/index.ts` against
`scripts/qa/smoke-test.ts` surfaced four gaps:

1. **Per-row PATCH/DELETE on `/api/clients/:id` and `/api/deals/:id`** —
   both routers use `router.use(requireAuth)`, so a refactor to per-route
   `requireAuth` that forgets one verb would ship unprotected. Only
   GET+POST were probed before today.
2. **`DELETE /api/properties/:id/save`** — the POST was probed on
   2026-05-02; the DELETE wasn't.
3. **`PATCH` and `DELETE /api/properties/:id/checklists/:checklistId`** —
   per-row mutations on a surface that only had its collection-level
   verbs probed.
4. **`POST /api/stripe/webhook` with no `stripe-signature` header** — the
   webhook is unauthenticated by design (Stripe signs with a shared
   secret instead of a Bearer token). The signature is the only thing
   stopping anyone from posting forged subscription events. It was never
   probed.

A small bonus: the bare `GET /api/properties/<bogus>` 404 was missing
(only `/walkscore` on a bogus id was probed), so the `:id` param
middleware's NOT_FOUND branch wasn't pinned.

## Files

- `scripts/qa/smoke-test.ts` — changelog entry + helper + 9 new probes.
- `apps/api/src/__tests__/daily-review-2026-05-03.test.ts` — pins the
  expanded probe list, the verb-parity audit, the Stripe webhook
  contract, the property `:id` param exit branches, and the
  update-deal `FELL_OUT` requirement.
