## Drift: CLAUDE.md vs reality

### Missing from CLAUDE.md (real, not documented)

These ten endpoints exist in the code but are absent from the CLAUDE.md API Endpoints table. They are not edge cases — they're real features in active use:

2. `POST /api/properties/geocode` — place ID → property resolution; used by the search flow.

5. `GET /api/properties/:id/checklists` — property checklists (read).
6. `POST /api/properties/:id/checklists` — create checklist.
7. `PATCH /api/properties/:id/checklists/:checklistId` — update.
8. `DELETE /api/properties/:id/checklists/:checklistId` — delete.
9. `POST /api/auth/sync-profile` — onboarding profile sync; needs `requireAuthOnly` (no DB profile required) — important for the auth flow.
10. `DELETE /api/auth/me` — account deletion.
