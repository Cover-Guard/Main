## Coverage gaps closed (5 new probes + 1 redundancy guard)

`Main/scripts/qa/smoke-test.ts` (614 lines):

1. **`HEAD /health` → 200** — pins that helmet/compression/morgan don't
   accidentally drop HEAD support, which monitoring services use for
   low-cost liveness checks. A future middleware refactor that registers
   something verb-specific (e.g. `app.use((req, res, next) => req.method
   === 'GET' ? handler : next())`) would silently break uptime probes.
2. **`GET /api/properties` → 404** (collection-root catch-all) — pins that
   the bare collection root has no handler. `propertiesRouter` is mounted
   at `/api/properties` but the router itself does not register `/`. A
   regression that adds `propertiesRouter.get('/', ...)` would change this
   contract.
3. **`GET /api/stripe/webhook` → 404** (POST-only catch-all) — the webhook
   is POST-only; a GET should hit the catch-all 404. The 2026-05-03 run
   pinned the `POST /api/stripe/webhook 400 (no signature)` contract, but
   a regression that registers a debug GET handler at the same path
   would silently 200 since the catch-all only fires when no route matches.
4. **`POST /api/properties/search` → 404** (GET-only catch-all) — search
   is GET-only and gated by the search rate limiter before the router.
   Express only matches the verb on the registered route, so a POST falls
   through to the catch-all 404 (NOT a 405). Pinning this catches a
   refactor that adds a POST search variant without updating the smoke
   suite.
5. **`HEAD /api/properties/:id/report.pdf` → 401** (property-id-bound) —
   pins that the verb-agnostic param middleware + `requireAuth` chain
   handles HEAD the same as GET. Express maps HEAD to the GET handler by
   default, so the same `requireAuth` middleware fires. A regression where
   `requireAuth` special-cases GET (e.g. `if (req.method === 'GET')`)
   would let HEAD probes leak resource existence — the response would be
   200 with no body, but the existence-check would have succeeded.

`Main/apps/api/src/__tests__/daily-review-2026-05-05.test.ts` (344 lines,
7 describe blocks, 17 it blocks):
