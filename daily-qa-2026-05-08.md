## Coverage gaps closed (6 new probes + extended file-integrity guard)

`Main/scripts/qa/smoke-test.ts` (1085 lines):

1. **`OPTIONS /api/clients` from allowed origin → 204, ACAO echoed
   (and not `*`)** — third mounted surface beyond `/api/properties/search`
   (2026-05-06) and `/api/auth/me` (2026-05-07). The cors() middleware
   is mounted globally today, but `/api/clients` is mounted via the
   `clients.ts` router file (separate from `properties.ts` and `auth.ts`),
   so a per-router cors override on the clients router would be
   invisible to the existing two probes. `/api/clients` was picked
   because it's an authed agent-only surface where a CSRF-via-CORS
   regression would leak client PII. **Per-router pin matrix is now
   3-of-9 routers (properties, auth, clients).**

2. **`GET /api/properties/:id/walkscore` Cache-Control includes
   `s-maxage=86400` and `public`** — pins the 24-hour CDN cache on
   walkscore. The walkscore upstream is rate-limited; the cache is the
   primary defense. The `forceRefresh=true` branch emits no-cache; this
   probe pins the default (cached) branch only. The probe also handles
   the 503 fallback gracefully (skip header assertion if status === 503
   so the probe doesn't false-fail when the upstream is down).

3. **`GET /api/properties/:id/insurance` Cache-Control includes
   `s-maxage=7200` and `public`** — pins the 2-hour CDN cache on the
   insurance estimate. The premium calculation is CPU-intensive
   (it recomputes from the risk profile, applies all carrier-specific
   modifiers, and sums coverages); without the CDN cache, every page
   load on a property detail would force a recompute.

4. **`GET /api/properties/:id/insurability` Cache-Control includes
   `s-maxage=7200` and `public`** — pins the 2-hour CDN cache on the
   insurability assessment. Insurability is a derived view over the
   same risk profile that drives /risk and /insurance, so the 2h TTL
   is parity with those two endpoints.

5. **`GET /api/properties/:id/carriers` Cache-Control includes
   `s-maxage=3600` (NOT 7200) and `public`** — pins the 1-hour CDN
   cache on active carriers. **The 1h TTL is intentionally shorter
   than insurance/risk/insurability (2h) and walkscore (24h)** because
   of the VA-01 carrier-exit alert SLA: the alert pipeline polls
   carrier-status feeds daily but UI must not show a carrier that
   exited >1h ago. Pinning the **3600** value specifically (not just
   `s-maxage>0`) catches a regression that bumps it to 7200 to "match"
   insurance, which would silently violate the SLA. The probe asserts
   the exact 3600 value AND the daily-review test adds a reverse-pin:
   the probe entry `s-maxage=7200` on carriers must NOT appear.

6. **`GET /api/properties/:id/risk?refresh=true` Cache-Control
   includes `no-store` AND `private`, NOT `s-maxage=`** — pins the
   forceRefresh=true code path on /risk that yesterday's probe did
   not cover. The handler branches on `req.query.refresh === 'true'`:
   the cached branch calls `setCacheHeaders(res, 7200, 600)` and the
   refresh branch calls `setNoCacheHeaders(res)` (which emits
   `private, no-cache, no-store, must-revalidate`). The refresh branch
   ALSO invalidates four dependent caches (insurance, carriers,
   insurability, publicData); a regression that drops setNoCacheHeaders
   would let the CDN serve a 7200s stale response after the caller
   explicitly asked for a fresh read. The probe pins all three
   discriminators: `no-store` present, `private` present, `s-maxage=`
   absent.
